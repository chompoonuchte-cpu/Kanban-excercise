# Product Spec — Kanban Board App


## แอปนี้คืออะไร

อยากได้แอปกระดาน Kanban ไว้จัดการงานสำหรับทีมเล็ก ๆ (ประมาณ 2–15 คน) หน้าตาเหมือน Trello แบบย่อ ๆ
มีกระดาน (board) แบ่งเป็นคอลัมน์ (เช่น To Do / Doing / Done) แล้วในแต่ละคอลัมน์มีการ์ดงานที่ลากย้ายไปมาได้

ใช้งานบนเว็บ (เปิดผ่าน browser) เป็น single-page app หน้าจอลื่น ๆ ไม่ต้องโหลดหน้าใหม่บ่อย ๆ

---

## ใครใช้

ทีมทำงานเล็ก ๆ ที่อยากเห็นภาพรวมว่างานไหนอยู่สถานะไหน ใครรับผิดชอบอะไร อะไรใกล้ครบกำหนด
ทุกคนในทีมต้องสมัคร/ล็อกอินก่อนถึงจะใช้ได้ ไม่ใช่แอปสาธารณะ

---

## ฟีเจอร์ที่ต้องมี

### 1. ผู้ใช้และการล็อกอิน
- สมัครสมาชิก / เข้าสู่ระบบ / ออกจากระบบ
- ต้องล็อกอินก่อนถึงจะเข้าหน้ากระดานได้ ถ้ายังไม่ล็อกอินให้เด้งไปหน้า login
- มีหน้าโปรไฟล์ของตัวเอง แก้ชื่อที่แสดง อีเมล และเปลี่ยนรหัสผ่านได้

### 2. กระดาน (Board)
- สร้างกระดานใหม่ได้ ตั้งชื่อได้ แก้ชื่อได้ ลบได้
- คนที่สร้างกระดานคือเจ้าของ มีสิทธิ์ลบได้ คนอื่นที่เป็นแค่สมาชิกลบไม่ได้
- เชิญคนอื่นเข้ามาเป็นสมาชิกของกระดานได้ (เชิญได้เฉพาะคนที่สมัครเป็น user แล้ว)

### 3. คอลัมน์ (Column)
- เพิ่ม / เปลี่ยนชื่อ / ลบคอลัมน์ได้
- เรียงลำดับคอลัมน์ได้
- ใส่สีให้คอลัมน์ได้ (เลือกจากสีที่เตรียมไว้ หรือกำหนดเอง หรือเอาออกให้กลับเป็นสีเทาเฉย ๆ) สีนี้ใช้ตกแต่งทั้งคอลัมน์ให้แยกแยะง่าย

### 4. การ์ดงาน (Card)
การ์ดหนึ่งใบแทนงานหนึ่งชิ้น ในการ์ดมี:
- **ชื่องาน** (จำเป็น) และ **คำอธิบาย** (ใส่หรือไม่ใส่ก็ได้)
- **วันครบกำหนด (due date)** เลือกจากปฏิทิน ถ้าเลยกำหนดแล้วให้เห็นชัดว่าเลยแล้ว
- **ป้ายกำกับ (label)** ติดได้หลายป้าย แต่ละป้ายมีชื่อกับสี สร้าง/แก้ชื่อ/แก้สีป้ายได้
  - ในจำนวนป้ายที่ติด ให้เลือกได้ว่าป้ายไหนเป็น "หมวดหมู่หลัก" ของการ์ด ป้ายหลักนี้จะโชว์เด่นอยู่บนหน้าการ์ด (ป้ายอื่น ๆ ไปจัดการในรายละเอียดการ์ด)
- **ผู้รับผิดชอบ (assignee)** มอบหมายได้หลายคนต่อหนึ่งการ์ด หน้าการ์ดโชว์รูปโปรไฟล์ของคนที่รับผิดชอบ
- **งานย่อย (subtask)** เป็น checklist ในการ์ด ติ๊กเสร็จ/ยังได้ เพิ่ม/แก้ชื่อ/เรียงลำดับ/ลบได้ และโชว์ความคืบหน้า (เช่น เสร็จ 3 จาก 5) บนหน้าการ์ด

### 5. ลากวางจัดการงาน (Drag & Drop)
- ลากการ์ดย้ายข้ามคอลัมน์ได้ (เปลี่ยนสถานะงาน)
- ลากสลับลำดับการ์ดในคอลัมน์เดียวกันได้
- ลากแล้วต้องลื่น ตำแหน่งที่ย้ายต้องถูกบันทึกจริง รีเฟรชหน้าแล้วยังอยู่ที่เดิม

### 6. เห็นความเปลี่ยนแปลงของเพื่อนร่วมทีม
- ถ้ามีหลายคนเปิดกระดานเดียวกัน ใครขยับการ์ดหรือแก้อะไร อีกคนควรเห็นตามในเวลาไม่นาน (ไม่ต้องเรียลไทม์เป๊ะ ๆ แค่อัปเดตให้เองเป็นระยะก็พอ)

---

## ข้อกำหนดทั่วไป (เท่าที่จำเป็น)

- ข้อมูลต้องถูกเก็บจริงในฐานข้อมูล ปิดแล้วเปิดใหม่/รีเฟรชต้องไม่หาย
- มีหน้าบ้าน (เว็บที่ผู้ใช้เห็น) และหลังบ้าน (เซิร์ฟเวอร์ + ฐานข้อมูล) แยกกัน คุยกันผ่าน API
- เวลากดทำอะไร (เช่น ย้ายการ์ด ติ๊ก subtask) หน้าจอควรตอบสนองทันที ไม่ต้องรอเซิร์ฟเวอร์ตอบก่อน ถ้าบังเอิญพลาดค่อยถอยกลับ
- กรอกข้อมูลผิด ๆ ควรมีการเตือน เช่น ชื่องานว่างไม่ได้ ชื่อยาวเกินไปไม่ได้ รหัสผ่านสั้นเกินไปไม่ได้

### ขอบเขตของข้อมูล (กันไว้ไม่ให้พัง)
- ชื่อกระดาน / ชื่อคอลัมน์: ห้ามว่าง ยาวไม่เกิน 100 ตัวอักษร
- ชื่อการ์ด: ห้ามว่าง ยาวไม่เกิน 255 ตัวอักษร
- คำอธิบายการ์ด: ไม่เกิน 5,000 ตัวอักษร
- ชื่องานย่อย: ห้ามว่าง ไม่เกิน 100 ตัวอักษร และมีได้ไม่เกิน 20 อันต่อการ์ด
- รหัสผ่านใหม่: อย่างน้อย 8 ตัวอักษร และต้องยืนยันให้ตรงกัน

---

## สิ่งที่ "ยังไม่ต้องทำ" ในเวอร์ชันแรก

เพื่อไม่ให้บานปลาย รอบแรกขอแค่นี้ก่อน — สิ่งเหล่านี้ไว้ค่อยคิดทีหลัง:
- ระบบแจ้งเตือน (notification / อีเมล)
- คอมเมนต์ในการ์ด / แนบไฟล์
- มุมมองแบบอื่น (ปฏิทิน, ไทม์ไลน์, รายงาน)
- บทบาท/สิทธิ์ละเอียด ๆ (มีแค่เจ้าของกับสมาชิกพอ)
- แอปมือถือ (เอาเว็บก่อน)

---

## แนวทางเทคโนโลยี 

- หน้าบ้าน: เว็บ SPA (เช่น React)
- หลังบ้าน: API + ฐานข้อมูล (เช่น Node.js + PostgreSQL)
- ล็อกอินด้วย token (เช่น JWT)
